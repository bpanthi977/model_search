* model_search

Train Neural Networks and explore hyperparameter space for tuning.

This project studies the use of *local surrogate models* to accelerate HPC applications. All training is driven by YAML config files and command line arguments; the structure of YAML configs is defined in =config.py=.

* Dependencies and setup

This project uses [[https://docs.astral.sh/uv/][uv]] to manage dependencies.
1. Install =uv=
2. Run =uv sync= in the project folder
3. Activate the venv before running the code =source .venv/bin/activate=

* Training

1. Create a config file. See =configs/hgf_model_structure.yaml= for example.
2. Run the main.py script:
   #+begin_src shell
   source .venv/bin/activate
   python main.py --config configs/hgf_model_structure.yaml
   #+end_src

   You will see a progress bar as training goes on.

3. The logs will be saved in =logs/hgf-20p/<date>-<time>-<random_string>/=

Override any config field on the command line:
#+begin_src shell
python main.py --config configs/hgf_model_structure.yaml --train.model.activation tanh --train.epoch 200
#+end_src

Resume training from a checkpoint:
#+begin_src shell
python main.py --checkpoint logs/hgf-20p/20260101-120000-AbCd/
#+end_src

Validate a checkpoint on the dataset:
#+begin_src shell
python main.py --validate --checkpoint logs/hgf-20p/20260101-120000-AbCd/
#+end_src

Export a TorchScript =model.pt= from a checkpoint:
#+begin_src shell
python main.py --create-model --checkpoint logs/hgf-20p/20260101-120000-AbCd/
#+end_src

* Hyperparameter Search

Bayesian hyperparameter tuning:
#+begin_src shell
python main.py --tune --config configs/hgf_model_structure.yaml
#+end_src

Grid search over hyperparameter combinations:
#+begin_src shell
python grid_search.py --config configs/hgf_model_structure.yaml --grid configs/hgf_model_structure_01.csv --wait --shared-memory
#+end_src

- =--shared-memory= shares the dataset tensor across training processes (less RAM demand).
- =--wait= blocks until all runs complete (useful for SLURM/PBS job submission).

* Visualize → Prune → Export

After training, check for dead neurons, prune them, and export:
#+begin_src shell
python visualize_activations.py --checkpoint logs/hgf-20p/20260101-120000-AbCd/ --dataset.subset -1
python prune_model.py --checkpoint logs/hgf-20p/20260101-120000-AbCd/ --threshold 1e-9
python main.py --create-model --checkpoint logs/hgf-20p/20260101-120000-AbCd_pruned/
#+end_src

* Code Organization

- =main.py= :: CLI entry point — dispatches to train, validate, tune, or export.
- =model.py= :: MLP model builder. Defines all layer types (Linear, NALU, MULT, Split, Join).
- =train.py= :: Optimizer setup, training loop, checkpoint saving. =train_log= saves loss curves, model, and metadata to the logs folder.
- =config.py= :: All config dataclasses. Defines what YAML fields are valid.
- =dataset.py= :: HDF5 dataset loader.
- =tune.py= :: Bayesian hyperparameter tuning via Optuna.
- =grid_search.py= :: Runs training across all combinations of hyperparameters specified by a grid CSV file. See =configs/hgf_model_structure_01.csv= for example.
- =visualize_activations.py= :: Captures post-activation outputs via forward hooks and plots them as violin plots. Used to identify dead neurons before pruning.
- =prune_model.py= :: Removes dead neurons from a trained checkpoint. Slices weight matrices and saves a new smaller checkpoint.
- =validate_model.py= :: Evaluates a checkpoint; generates error histograms.
- =test_evd_model.py= :: Detailed analysis of EVD models: Jacobian heatmaps and input sweep plots.
- =lr_tune.py= :: LR range test — sweeps LR from 1e-6 to 10 and plots loss.
- =visualize.py= :: Plots loss curves and weight matrices.
- =energy.py= :: Visualizes LULESH energy output differences.
- =logs.py= :: Utilities to read and compare multiple training run logs.
- =utils.py= :: Loss functions and hyperparameter extraction helpers.
